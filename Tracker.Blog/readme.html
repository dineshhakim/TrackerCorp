Task

Add the ability to post comments to individual blog posts.

Requirements

  1. Create a table for comments
  2. Add a comment model
  3. A comment should include: the user name, when it was posted, and the message.
  4. When a user is looking at the details view of a blog (~/posts/details/#), all comments regarding that post should be displayed.
  5. The user should also have an input form to post a comment regarding the post. There should be atleast a field for their username and their comment message.
  6. Using JavaScript, display the remaining characters a user can enter their message.
  7. Using JavaScript, do something distinct to show the comment just posted by the user.
  8. The max length of the comment should be 500 characters.


Notes

  • Currently, EF is set to rebuild the database anytime you change a model.
  • You do not need to verify the identity of the person posting the comment. Anyone can comment under any name and do not need to be logged into do so.
  • You can use any resources you need to complete the task: google, msdn, etc.
  • The styling of the comments is up to you.

Server Side Requirements

  • .NET 4.5.2
  • Entity Framework 6.1.3
  • ASP.NET MVC 5.2.3
  • LocalDB

Client Side Components

  • jQuery 2.1.4
  • jQuery UI 1.11.4
  • Moderizr 2.8.3
